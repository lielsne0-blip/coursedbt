# 🔄 זרימת העבודה הקבועה ב-Git

המתכון הקבוע לכל שינוי. כל הפקודות רצות מתוך תיקיית הפרויקט:

```bash
cd "/Users/liel.sne/Desktop/corse dbt/jaffle_shop"
```

---

## 1️⃣ להתחיל מ-main מעודכן
```bash
git checkout main
git pull origin main
```
מוודא שמתחילים מהגרסה הכי עדכנית.

## 2️⃣ ליצור ברנץ' חדש לשינוי
```bash
git checkout -b feature/שם-תיאורי
```
כל שינוי חדש = ברנץ' חדש. לדוגמה: `feature/add-orders-model`

## 3️⃣ לעשות את השינויים בקבצים
עורכים / מוסיפים / מוחקים קבצים כרגיל בעורך.

## 4️⃣ לבדוק מה השתנה (מומלץ)
```bash
git status
```

## 5️⃣ להוסיף את כל השינויים
```bash
git add -A
```
ה-`-A` תופס הכל: קבצים חדשים, מחיקות ושינויים.

## 6️⃣ לעשות commit עם הודעה
```bash
git commit -m "תיאור קצר של מה שעשית"
```
שומר את השינוי **מקומית** (על המחשב שלך).

## 7️⃣ לדחוף ל-GitHub
```bash
git push -u origin feature/שם-תיאורי
```
מעלה את הברנץ' ל-GitHub. **רק עכשיו** הוא יופיע שם.

## 8️⃣ לפתוח Pull Request
להיכנס לקישור שמופיע בטרמינל (או ל-GitHub) וללחוץ **Create pull request**.

## 9️⃣ למזג (Merge)
בעמוד ה-PR ללחוץ **Merge pull request**.

## 🔟 לחזור ל-main מעודכן
```bash
git checkout main
git pull origin main
```
מסיים את המעגל — חוזרים להתחלה מוכנים לשינוי הבא.

---

## 💡 הכלל הכי חשוב לזכור

| פקודה | מה היא עושה | איפה |
|-------|-------------|------|
| `commit` | שומר את השינוי | 💻 רק אצלך |
| `push` | מעלה ל-GitHub | ☁️ בשרת |

➡️ **בלי `push` — אף אחד ב-GitHub לא רואה את הברנץ' או השינוי.**

---

## 🛠️ פתרון תקלות נפוצות

- **`fatal: not a git repository`** — את לא בתיקייה הנכונה. הריצי קודם:
  `cd "/Users/liel.sne/Desktop/corse dbt/jaffle_shop"`
- **לא רואה את הברנץ' ב-GitHub** — כנראה לא עשית `git push`. ברנץ' מקומי לא קיים בשרת עד שדוחפים אותו.
- **`Authentication failed` / `403`** — בעיית טוקן. צריך טוקן עם הרשאת `repo`, ולהזין אותו כ-Password (לא הסיסמה הרגילה).
